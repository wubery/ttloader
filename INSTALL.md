# Установка Video Poster на сервер

Панель автопостинга в TikTok/YouTube Shorts в Docker. Панель закрыта входом
(логин/пароль, опционально — подтверждение через Telegram); защита на уровне приложения.

## Требования

- Linux-сервер с **Docker** и **docker compose** (плагин v2), **git**.
- Открытый порт панели (по умолчанию **8088**).

## Установка (рекомендуется — из git, чтобы работало самообновление)

```bash
git clone https://github.com/wubery/ttloader.git video-poster
cd video-poster
bash install.sh
```

Установщик соберёт контейнеры, настроит хостовый апдейтер (systemd `vp-updater`)
и **один раз** покажет в консоли логин/пароль:

```
Панель:  http://<IP-сервера>:8088
Логин:   admin
Пароль:  <случайный пароль>
```

Первая сборка — несколько минут (тянется образ Playwright ~2 ГБ и Chromium).

## Параметры установки

```bash
VP_PORT=9000 bash install.sh          # другой порт
VP_USER=boss bash install.sh          # другой логин
VP_PASS='МойПароль' bash install.sh   # задать свой пароль
VP_REGEN=1 bash install.sh            # пересоздать .env с новыми логином/паролем
```

Логин/пароль хранятся в `.env` (в git не попадает) и применяются при первом запуске.
Дальше пароль меняется в самой панели → вкладка **«Настройки»**.

## Вход и Telegram

- Вход в панель — логин/пароль на странице входа.
- В «Настройках» можно задать **токен бота** (@BotFather) и **chat_id**, включить
  **вход через Telegram** (панель пришлёт код в чат) и получать **уведомления**
  о постинге и упавших прокси. Бот-команды: `/queue`, `/accounts`; присланное боту
  видео добавляется в библиотеку.

## Управление

```bash
cd video-poster
docker compose ps
docker compose logs -f backend
docker compose restart
docker compose down
```

## Обновление

- Из панели: «Настройки» → **«Обновить с GitHub»** (хостовый `vp-updater` сделает
  `git pull` + пересборку). Работает, только если проект **склонирован через git**.
- Вручную: `git pull && docker compose up -d --build`.

## Важно

- Данные (аккаунты, куки, видео, БД) — в docker-томе `vp_data`, при обновлении сохраняются.
- Прокси и куки аккаунтов настраиваются в панели. Вход в аккаунт — логин/пароль +
  код с почты (через прокси аккаунта); при капче панель покажет скриншот, тогда войдите
  в антидетект-браузере через тот же прокси и импортируйте куки (кнопка «Импорт кук»).
- SOCKS5 с логином/паролем Chromium не поддерживает — используйте `http://`-прокси.
- Если прокси на том же VPS, что и панель, IP входа и постинга совпадут (это хорошо для TikTok).
