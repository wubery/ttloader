# Video Poster

Самостоятельная (self-hosted) панель автопостинга видео в **TikTok** и **YouTube Shorts**
с авторизацией по кукам, личным прокси на аккаунт и вжиганием собственного баннера
(водяного знака) в видео.

## Возможности

- **Авторизация по кукам** (Playwright `storage_state`) — без ввода пароля, куки
  импортируются через панель (экспорт из браузера).
- **Личный прокси на каждый аккаунт** — постоянный IP: прокси подставляется при
  каждом запуске браузера (`browser.launch(proxy=...)`).
- **Панель управления**: аккаунты, видео, баннеры, очередь постинга со статусами и логами.
- **Баннер** — статичная картинка (PNG с прозрачностью) или **зацикленное видео**.
  Позиция и размер настраиваются мышью в **превью видео** (вкладка «Баннер+превью»),
  хранятся в долях кадра и «вжигаются» в ролик через ffmpeg — украсть видео без баннера нельзя.
- **Отложенный постинг** — время публикации задаётся при создании поста (APScheduler).

## Архитектура

- `backend/` — FastAPI + SQLAlchemy (SQLite) + APScheduler; ffmpeg для баннера;
  Playwright (Chromium) для загрузки в соцсети. Загрузчики: `app/services/uploaders/`.
- `frontend/` — React + Vite + TypeScript, отдаётся через nginx, который проксирует `/api` на бэкенд.
- Данные (видео, баннеры, куки, БД) — в docker-томе `vp_data` (`/data` внутри контейнера).

## Запуск в Docker (домашний сервер)

```bash
docker compose up -d --build
```

Панель: **http://<адрес-сервера>:8088**

Проверка здоровья: `http://<адрес-сервера>:8088/api/health` (покажет доступность ffmpeg и Playwright).

## Как пользоваться

1. **Аккаунты** → создать аккаунт (TikTok/YouTube), указать прокси `http://user:pass@host:port`.
2. Импортировать **куки**: в браузере, где вы залогинены, экспортируйте `storage_state`
   (расширение вроде *Get cookies.txt* даёт массив cookies — тоже принимается; либо
   Playwright storage_state JSON). Загрузите файл в карточке аккаунта.
3. **Видео** → загрузить ролики. **Баннеры** → загрузить свой баннер.
4. **Баннер+превью** → выбрать видео и баннер, перетащить/отмасштабировать, «Сохранить позицию».
5. **Новый пост** → аккаунт + видео + баннер + подпись + (опц.) время → «Поставить в очередь».
6. **Очередь** → статусы (`ожидает → рендер → постинг → готово`), логи, повтор при ошибке.

## Важно про селекторы

TikTok/YouTube периодически меняют вёрстку. CSS-селекторы вынесены в начало файлов
`backend/app/services/uploaders/tiktok.py` и `youtube.py` — при поломке постинга
правьте их там. Для отладки поставьте `HEADLESS=false` (нужен графический сервер).

## Экспорт кук (быстрый способ)

Установите расширение, экспортирующее cookies в JSON, залогиньтесь в TikTok/YouTube,
экспортируйте cookies для домена и загрузите файл в панель. Панель принимает как
массив `[{name,value,domain,...}]`, так и полный `storage_state`.
